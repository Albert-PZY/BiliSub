# 隐私政策草案

最后更新：2026-05-05

本隐私政策适用于浏览器扩展《B站字幕工作台》。

## 1. 我们处理哪些数据

扩展本身会处理以下几类信息：

- 用户在设置页手动输入的 API Key
- 用户选择的摘要 provider、model、温度、输出长度等配置
- 当前 B 站视频页面的基本上下文，例如视频链接、BV 号、标题
- 当前视频对应的 AI 字幕内容

## 2. 数据如何存储

- API Key 和摘要设置仅保存在当前浏览器本地的 `chrome.storage.local`
- 这些数据不会自动同步到 `chrome.storage.sync`
- 扩展不会将 API Key 上传到扩展作者自建服务器

## 3. 何时会向第三方发送数据

默认情况下，扩展不会向任何 LLM 服务发送字幕内容。

只有在满足以下条件时，扩展才会向用户当前选择的官方 LLM provider 发送请求：

- 用户已经在设置页填写 API Key 和 model
- 用户已经执行“测试连接”并通过
- 用户在侧边栏中主动点击“生成摘要”

在这种情况下，请求可能包含：

- 字幕文本
- 视频标题
- 用户在本地配置的 model 参数

第三方接收方仅限用户当前选择的官方 provider：

- OpenAI：`https://api.openai.com`
- 阿里云百炼：`https://dashscope.aliyuncs.com`

## 4. 我们不会做什么

- 不会把 API Key 上传到扩展作者的服务器
- 不会在用户未触发摘要时把字幕内容发送到 OpenAI 或阿里云百炼
- 不会依赖 `localhost`、`127.0.0.1`、本地 Python 或本地伴生服务
- 不会向未在产品内声明的第三方域名发送摘要请求

## 5. B 站登录态的使用方式

扩展会复用用户当前浏览器中的 B 站页面登录态，以便访问当前视频可见的字幕资源。

扩展不会导出、展示或单独持久化 B 站账号密码。

## 6. 用户可如何控制数据

用户可以随时：

- 在设置页修改或清空 API Key
- 删除浏览器中的扩展本地数据
- 卸载扩展

## 7. 第三方服务条款

当用户主动使用摘要功能时，相关请求还将受第三方 provider 自身的服务条款和隐私政策约束。用户应自行确认所选 provider 的合规要求。
